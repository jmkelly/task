Why Task

One workflow surface for developers, teams, and AI agents.

Task is designed around explicit workflows instead of opaque SaaS behavior. The UX mirrors the current Kanban board: calm surfaces, strong hierarchy, practical states, and minimal visual noise.

Developer-first

CLI that fits automation and daily terminal work

Use plain text for people, JSON for scripts, and consistent commands for local or remote environments.

Visual clarity

Kanban board with explicit task states

Move from todo to in progress, blocked, and done with UI patterns built for focus rather than decoration.

Integration-ready

API surface for bots, agents, and custom tooling

Connect automations, notifications, or AI workflows without giving up ownership of your task data.

Core value

Built for practical task operations, not a marketing demo.

Self-hosted storage

SQLite persistence, schema migration support, and no vendor lock-in.

Searchable work

Full-text and semantic search help humans and agents recover context fast.

Structured outputs

Machine-readable JSON and explicit error handling make automations resilient.

Modern workflow states

Track blocked work, assignees, due dates, tags, and project context without clutter.

Quick start

Get a local workflow running in a few commands.

01

Start the local API

task server start
task server status
02

Point the CLI at your API

task config set api-url http://localhost:8080
task --help
03

Start capturing work

task add "Review pull request"
task list --status todo
task search "review"
Run from source .NET 
dotnet build Task.Cli/Task.Cli.csproj -c Release
dotnet run --project Task.Api
Publish your own binary Release 
dotnet publish Task.Cli -c Release -r linux-x64 --self-contained -p:PublishSingleFile=true
dotnet publish Task.Cli -c Release -r win-x64 --self-contained -p:PublishSingleFile=true

Integrations and advanced usage

Task is strongest when it becomes part of your toolchain.

Use the same task model across scripts, internal services, automation jobs, and browser workflows.

API workflows

Automate task creation, filtering, and completion

REST endpoints and OpenAPI documentation make it straightforward to plug Task into services and internal dashboards.

Agent operations

Give coding agents a durable place to read and update work

Task’s explicit statuses, tags, project fields, and JSON output are well suited to LLM-driven execution loops.

Notifications

Extend with providers like Telegram and custom hooks

Build lightweight operational awareness without adding another closed workflow system to your stack.

Typical use cases

Coordinate work across local development, API consumers, and visual planning.

  • Track implementation work from the CLI while keeping stakeholders aligned in the board view.
  • Drive background jobs or AI agents with structured task metadata and predictable status transitions.
  • Keep private, portable task data in your own environment instead of a hosted black box.
